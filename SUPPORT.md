# Support

Thanks for using Tabnas!

## Where to ask

- **Bug reports & feature requests** — open an issue on the relevant
  repository using the issue forms. If you're not sure which repository,
  use [tabnas/parser](https://github.com/tabnas/parser/issues).
- **Questions & usage help** — open an issue with the *documentation*
  template; questions often reveal documentation gaps we want to fix.
- **Security issues** — never a public issue; see
  [SECURITY.md](SECURITY.md).

## Before filing

1. Check the repository README and existing issues (including closed ones).
2. Note the package version and which implementation you're using —
   TypeScript (`@tabnas/*`) or Go (`github.com/tabnas/*/go`).
3. A minimal reproduction (ideally a failing test case) gets issues fixed
   dramatically faster.

## Response expectations

Tabnas is maintained on a best-effort basis. Issues are triaged regularly,
but there is no guaranteed response time. Well-written reports with
reproductions are prioritized.
